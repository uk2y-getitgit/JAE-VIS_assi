# 🤖 윈도우 AI 비서 (JARVIS형) — Claude Code 프로젝트 지침서

---

## 📌 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | Windows AI Assistant (JARVIS Style) |
| 목적 | 개인용 윈도우 상주 AI 비서 (일정·메모 관리 중심) |
| 형태 | 트레이 아이콘 + 바탕화면 마스코트 캐릭터 위젯 |
| 개발 환경 | VSCode + Claude Code / Windows Desktop |
| 최종 배포 | 설치형 (.exe 인스톨러) |
| 테스트 방식 | 개발 중 실행형(.exe) 직접 실행 |

---

## 🎨 디자인 컨셉

- **스타일**: JARVIS / Iron Man 테마 — SF 홀로그램 AI 비서 이미지
- **색상**: 딥 네이비·블랙 배경 + 시안(#00CFFF)·전기 블루 형광 강조색
- **캐릭터**: 홀로그램 느낌의 AI 마스코트 (원형 에너지 오브 또는 아이언맨 AI 스타일 아이콘)
- **UI 톤**: 심플·위젯형, HUD(헤드업디스플레이) 느낌의 미니멀 패널
- **폰트**: 미래적 고딕 계열 (예: Orbitron, Exo 2, Rajdhani)
- **애니메이션**: 캐릭터 idle 애니메이션(빛 맥동), 말풍선 페이드인, 패널 슬라이드

## 👤 사용자 시스템

### 로그인 구조
- 앱 최초 실행 시 **사용자 선택 화면** 표시
- 사용자 식별: **이름 + 4자리 PIN**
- 예시: `사용자1 / 4040`
- 다중 사용자 지원 (각 사용자별 독립 데이터 저장)
- 로그인 후 해당 사용자의 메모·일정 데이터만 로드

### 데이터 저장 원칙
- 사용자별 데이터는 **로컬 파일로 분리 저장** (예: `data/users/{username}/`)
- 앱 업데이트·재설치 시 기존 데이터 유지 (데이터 폴더 별도 관리)
- 저장 형식: JSON 또는 SQLite

## 🖥️ UI 구성요소

### 1. 트레이 아이콘
- 윈도우 시스템 트레이에 상주
- 우클릭 메뉴:
  - 메인 패널 열기
  - 빠른 메모
  - 오늘 일정 보기
  - 자동실행 ON/OFF 토글
  - 종료
- 알람 발생 시 트레이 아이콘 깜빡임

### 2. 바탕화면 마스코트 캐릭터 위젯
- 바탕화면에 항상 표시 (Always on Desktop, but NOT always on top)
- 마우스로 드래그하여 위치 이동 가능
- 위치 기억 (앱 재시작 시 마지막 위치 유지)
- **Idle 상태**: 홀로그램 맥동 애니메이션
- **알림 상태**: 말풍선 팝업 표시
  - 예: "오늘 일정 브리핑 준비됐어요 ▶"
  - 말풍선 클릭 → 일정 리스트 팝업 (텍스트 복사 가능)
- **클릭**: 메인 패널 열기

### 3. 메인 패널 (단축키 호출 포함)
단축키로 3가지 화면 간 **스왑(토글)** 가능:

| 단축키 | 화면 |
|--------|------|
| `Ctrl + Shift + N` | 빠른 메모 입력창 |
| `Ctrl + Shift + S` | 오늘 일정 요약 |
| `Ctrl + Shift + M` | 메인 기능 패널 전체 |

> 단축키는 추후 사용자 설정에서 변경 가능하도록 설계

## 📅 일정 관리 기능

### 일정 입력 필드
| 필드 | 설명 |
|------|------|
| PJT명 (프로젝트명) | 일정이 속한 프로젝트 이름 |
| 마감기한 | 날짜·시간 선택 (1주 전 자동 알림 발생) |
| 작업 예정일 | 실제 작업할 날짜 (오늘 일정 기준) |
| 메모 | 해당 일정 관련 간단한 노트 |

### 달력 뷰
- 월간 달력 뷰 기본 제공
- 날짜 클릭 시 해당 날의 일정 목록 표시
- 마감기한 D-7 이내 항목은 색상 강조 표시

### 알림 시스템
- **마감 1주 전 알림**: 마감기한 기준 7일 전 자동 발생
- 알림 방식: 캐릭터 말풍선 팝업
- 말풍선 내용 예시:
  ```
  ⚠️ [웹사이트 리뉴얼] 마감까지 7일 남았어요
  마감: 2025-06-15 | 작업예정: 오늘
  ```
- 말풍선 클릭 → 해당 일정 상세 리스트 표시
- 리스트 내 텍스트 **복사 버튼** 제공

### 오늘 일정 브리핑
- 매일 아침 (설정 시간, 기본 09:00) 자동 말풍선 팝업
- 오늘 **작업 예정일**로 등록된 일정 목록 표시
- 내용 예시:
  ```
  📋 오늘의 일정 (5월 8일)
  · [웹사이트 리뉴얼] 기획안 작성
  · [앱 개발] UI 시안 검토
  ```

## 🚀 개발 단계 (권장 순서)

1. **Phase 1**: 프로젝트 세팅 + 로그인 화면 (이름·PIN)
2. **Phase 2**: 트레이 아이콘 + 기본 메뉴
3. **Phase 3**: 바탕화면 마스코트 캐릭터 위젯 (드래그 이동)
4. **Phase 4**: 메인 패널 UI (JARVIS 테마)
5. **Phase 5**: 메모 기능 구현
6. **Phase 6**: 일정 관리 + 달력 뷰 구현
7. **Phase 7**: 알람·말풍선 알림 시스템
8. **Phase 8**: 단축키 스왑 기능
9. **Phase 9**: 자동실행 설정·전체 설정 화면
10. **Phase 10**: electron-builder로 설치형 .exe 패키징

## 📋 Claude Code 작업 시 주의사항

- 각 Phase 완료 후 실행 테스트 먼저 진행
- 데이터 저장 구조는 Phase 1에서 확정 후 변경 최소화
- 사용자 PIN은 평문 저장 금지 → bcrypt 또는 SHA-256 해시 처리
- 캐릭터 위치, 설정값은 앱 종료 시 자동 저장
- 코드 변경 시 기존 `data/` 폴더 내용은 절대 삭제하지 않도록 주의
- 윈도우 경로 구분자 (`\`) 처리 주의 (path.join 사용 권장)

## 🔧 Serena MCP 활용 지침

이 프로젝트는 **Serena**가 `mymy_ai_planner`로 활성화되어 있습니다.
코드 작업 시 단순 파일 읽기/쓰기 대신 **Serena 도구를 우선 사용**하세요.

### 언제 어떤 도구를 쓸지

| 작업 | 사용할 Serena 도구 |
|------|-------------------|
| 파일/폴더 구조 파악 | `list_dir`, `find_file` |
| 심볼(함수·변수·클래스) 위치 찾기 | `find_symbol`, `find_declaration` |
| 함수 구현체 탐색 | `find_implementations` |
| 특정 심볼을 참조하는 곳 찾기 | `find_referencing_symbols` |
| 파일 내 전체 심볼 구조 파악 | `get_symbols_overview` |
| 패턴/키워드 검색 | `search_for_pattern` |
| 함수·메서드 본문 교체 | `replace_symbol_body` |
| 심볼 앞/뒤에 코드 삽입 | `insert_before_symbol`, `insert_after_symbol` |
| 심볼 안전 삭제 | `safe_delete_symbol` |
| 심볼 이름 변경 (전체 참조 포함) | `rename_symbol` |
| 파일 내 텍스트 치환 | `replace_content` |
| 파일 읽기 | `read_file` |
| 새 파일 생성 | `create_text_file` |
| 진단(에러·경고) 확인 | `get_diagnostics_for_file` |
| 셸 명령 실행 | `execute_shell_command` |

### 작업 흐름 원칙

1. **탐색 우선**: 코드를 수정하기 전에 `get_symbols_overview` 또는 `find_symbol`로 구조를 먼저 파악한다.
2. **심볼 단위 수정**: 함수 전체를 교체할 때는 `replace_symbol_body`를 사용해 범위를 명확히 한다.
3. **참조 확인 후 삭제**: 심볼 삭제 전 `find_referencing_symbols`로 영향 범위를 확인한다.
4. **진단 확인**: 코드 작성 후 `get_diagnostics_for_file`로 오류·경고를 점검한다.
5. **셸 명령**: `npm start`, `npm install` 등 셸 명령은 `execute_shell_command`로 실행한다.

### 주의사항

- Serena 작업 시작 전 `initial_instructions`로 매뉴얼을 확인한다.
- `replace_symbol_body`는 심볼이 정확히 인덱싱된 경우에만 동작한다 — 새 파일은 `create_text_file` 후 재인덱싱을 기다린다.
- 한국어 경로(`시스템운영-절대지우지마시오(26년)`)가 포함되어 있으므로 셸 명령 실행 시 경로를 따옴표로 감싼다.
